📐 泊松分布模型说明

P(X=k) = (λ^k × e^(-λ)) / k!
P(主 k, 客 m) = P_home(k) × P_away(m) (假设主客独立)

⚠️ 局限: 真实比赛主客进球不独立 (强队进球更多+对手少攻)。修正: Dixon-Coles 模型对低比分调整, 我们当前是基础泊松。

🎯 自审结论 + 改进建议

🔴 严重缺口 (影响比分命中 23% 偏低的核心原因):
  1. 基础泊松 → Dixon-Coles: 当前模型假设主客进球独立, 真实比赛 0-0 / 1-1 / 1-0 / 0-1 概率被低估。修正: Dixon-Coles 对低比分 (0-0, 1-0, 0-1, 1-1) 加权 +5~10%, 对高比分 (3-3, 4-4) 降权 -5%。
  2. 门将扑救率 (Save %): 当前 GK 评分只算身价, 库尔图瓦 80% 扑救率 vs 普通门将 65% 差距巨大, 直接影响比分。
  3. 战术风格匹配: 控球队 vs 反击队 / 高位防线 vs 长传冲吊 这种"系统性相克"完全没建模, 是 7/7 比利时 4-1 美国 (战术碾压) 的关键。
🟡 中等缺口 (战术修正 v2 弥补, 但 v3 算法本身缺):
  1. 近期战绩/士气: 黄金一代最后一届战意、挪威 32 强爆冷巴西的士气、葡萄牙 PK 西复仇战意 — 这些都是 v3 算法缺的关键非量化信号。
  2. 伤停 (赛前预期): 萨拉赫伤、科尔多瓦伤、J 罗 35 岁体能 — 算法应该赛前就纳入, 而不是用红牌 (已踢) 间接表达。
  3. 历史交锋: 比利时 2014 2-1 美国 加时淘汰 (美国 16 强门槛宿命), 巴西 vs 挪威历史从未输 — 这些心理优势没建模。
  4. 裁判执法风格: 主裁判罚牌倾向, 严格 vs 宽松, 影响红黄牌数 + 比赛节奏。
🟢 算法已经做得好的 (12 项已纳入):
  1. 4 维评分 (FW/MID/DEF/GK) — 这是核心
  2. 教练战术 + 阵型
  3. 海拔 + 温度
  4. KO 阶段保守调整 (λ × 0.85)
  5. 加时赛概率
  6. 红牌惩罚 (跨场)
  7. 持球率档位
  8. 阵容深度惩罚
  9. 球员/教练占比
  10. 平局下限
  11. λ 上限 (4.5)
  12. FIFA 排名加权

💡 改进优先级建议

如果要做 v4 算法升级, 按 ROI 排:
  1. P0 (必须): Dixon-Coles 模型 — 直接提升比分命中率 (23% → 35-40%), 代码改动小 (泊松公式加 adjustment)
  2. P0 (必须): 门将扑救率 — 加 Save % 字段到 1248 球员主表, 喂到 GK 评分, 影响巨大
  3. P1 (高): 战术风格矩阵 — 给每队打"风格标签" (控球/反击/高位防线/长传冲吊), 加风格相克系数
  4. P1 (高): 赛前伤停名单 — 维护 26 人名单 + 赛前状态, 赛前更新 GK/FW/MID 评分
  5. P2 (中): 近期战绩加权 — 加权最近 3 场进球率/失球率, 反映黑马+低谷期
  6. P3 (低): 裁判执法风格 — 影响红黄牌数, 但 KO 阶段红牌惩罚已经做了
